Hello HTML5

HTML5 introduces a simple and more flexible way to write web documents. Here is an example of a basic HTML5 document:

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Hello HTML5 World</title>
</head>
<body>
    <h1>Hello HTML5</h1>
    <p>Welcome to the future of markup!</p>
</body>
</html>

The <!DOCTYPE html> declaration is simplified and the document is checked for conformance to specifications instead of traditional validation.

Loose Syntax

HTML5 allows a degree of syntax variability. Here is an example with optional elements and inferred close tags:

<!DOCTYPE html>
<meta charset="UTF-8">
<title>HTML5 Tag Soup Test</title>
<h1 title="more sloppy markup ahead!">HTML5</h1>
<p id=p1>Back to the future of loose markup!? 
<p>Yes it looks that way 
<ul>
    <li>optional elements 
    <LI>case is no problem 
    <li id=noquotes>quotes optional in many cases 
    <li>inferred close tags 
</ul>
<p>Oh my

Document Structure Changes

HTML5 introduces new structural elements like <header>, <footer>, and <section>:

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>HTML5 header and footer example</title>
</head>
<body>
    <header>
        <h1>Welcome to the Future World of HTML5.</h1>
        <h2>Don't be scared it isn't that hard!</h2>
    </header>
    <p>Some body content here.</p>
    <p>Some more body content here.</p>
    <footer>
        <p>Content of this example is not under copyright.</p>
    </footer>
</body>
</html>

Client-Side Graphics with <canvas>

The <canvas> element allows for rendering graphics on the client side:

<canvas id="canvas" width="300" height="300">
    <strong>Canvas Supporting Browser Required</strong>
</canvas>

<script>
    window.onload = function() {
        var canvas = document.getElementById("canvas");
        var context = canvas.getContext("2d");
        context.strokeStyle = "blue";
        context.strokeRect(10, 10, 150, 50);
        context.fillStyle = "rgba(218, 0, 0, 0.4)";
        context.fillRect(150, 30, 75, 75);
    }
</script>
